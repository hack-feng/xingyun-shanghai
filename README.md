# 上海城市展览厅交互式智能讲解助手

## 项目简介

这是一个基于Web技术的大屏展示系统，专为上海城市展览厅设计。系统采用历史、现代与未来三种视觉风格碰撞的设计理念，通过智能数字人讲解员，为参观者提供沉浸式的上海城市历史与文化体验。系统集成了魔珐星云数字人SDK和豆包大模型API，实现了智能问答和流式讲解功能。

## 功能特点

### 核心功能
- **大屏单屏展示**：专为大型显示器设计，所有内容在一屏内完整展示
- **智能数字人讲解**：集成魔珐星云SDK，提供逼真的数字人讲解服务
- **流式语音输出**：支持流式调用speak方法，实现流畅的语音讲解
- **逐句讲解**：智能分割文本，逐句进行讲解，字幕同步显示
- **智能问答**：集成豆包大模型API，支持用户提问并获取智能回答
- **结束讲解**：提供"结束讲解"按钮，可随时停止当前讲解

### 交互体验
- **历史碰撞风格**：融合历史、现代与未来三种视觉风格
- **交互式内容切换**：支持历史、现代、未来三个时间段的内容切换
- **丰富动画效果**：包括按钮涟漪、内容过渡、呼吸灯、脉冲、悬浮等多种动画
- **响应式交互**：所有交互元素都有即时视觉反馈
- **粒子背景**：动态粒子背景增强视觉效果

## 技术栈

- **前端技术**：HTML5, CSS3, JavaScript (ES6+)
- **UI框架**：原生CSS Grid和Flexbox布局
- **动画效果**：CSS3动画和过渡
- **数字人SDK**：魔珐星云Xingyun3D SDK (xmovAvatar@latest.js)
- **大模型API**：豆包大模型API (Volces)
- **流式处理**：Fetch API with ReadableStream

## 快速开始

### 环境要求

- 现代浏览器（Chrome 90+, Firefox 88+, Safari 14+, Edge 90+）
- 本地HTTP服务器（用于正确加载资源）

### 安装与运行

1. 克隆或下载项目文件到本地目录
2. 启动本地HTTP服务器（推荐使用Python）：
   ```bash
   # 在项目目录下执行
   npm install
   node server.js
   ```
3. 在浏览器中访问：`http://localhost:3000`

### 配置说明

在`config.js`文件中配置魔珐星云SDK和豆包API参数：

```javascript
// 魔珐星云SDK配置
const xingyunConfig = {
    appId: "your_app_id",
    appSecret: "your_app_secret",
    gatewayServer: "https://nebula-agent.xingyun3d.com/user/v1/ttsa/session"
};

// 豆包API配置
const doubaoConfig = {
    apiKey: "your_api_key",
    apiUrl: "https://ark.cn-beijing.volces.com/api/v3/chat/completions",
    model: "your_model_id"
};
```

## 使用说明

### 基本操作

1. **页面加载**：打开页面后，系统会自动初始化数字人讲解员
2. **内容切换**：点击顶部的"历史"、"现代"、"未来"按钮切换不同时间段的内容
3. **智能讲解**：切换时间段时，数字人会自动讲解对应内容
4. **字幕显示**：讲解内容会实时显示在左侧字幕区域
5. **结束讲解**：讲解过程中，点击字幕区域右下角的"⏹️ 结束讲解"按钮可停止当前讲解

### 交互元素

- **时间段按钮**：点击切换内容，带有涟漪效果和发光动画
- **内容卡片**：悬停时有上浮效果和阴影变化
- **高亮项目**：每个历史/现代/未来亮点都有悬浮动画效果，点击可触发详细讲解
- **问题输入框**：在底部输入问题，点击"提问"按钮，数字人会讲解豆包API返回的答案

## 注意事项

1. **本地服务器**：由于浏览器安全限制，必须通过HTTP服务器访问，不能直接打开HTML文件
2. **API密钥**：请妥善保管API密钥，不要提交到版本控制系统
3. **网络连接**：系统需要连接互联网才能加载SDK和调用API
4. **HTTPS**：生产环境建议使用HTTPS协议
5. **跨域问题**：确保API服务器支持CORS或配置代理
